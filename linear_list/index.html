数据结构之线性表

线性表是由相同类型的数据元素组成的有限序列。就像一条线一样，除了端点外，每个元素都有唯一的直接前驱和直接后继。

线性表有两种主要的存储方式：

1. 顺序存储（顺序表）

顺序存储中元素的存储是连续的。使用数组 data[] 存储数据，最大空间为 Maxsize，实际存储的元素个数为 length。顺序存储的优点是可以通过下标快速访问元素。

顺序存储可以分为静态分配和动态分配两类。静态分配使用数组静态分配存储空间，而动态分配则使用一段连续的存储空间的首地址存储数据。无论哪种方式，顺序表的插入和删除操作会导致数据的成片移动，不是很方便。

2. 链式存储（链表）

链式存储中元素的存储不是连续的，每个数据点叫做节点。为了遍历所有的节点，我们使用指针存储下一个节点的地址。一个节点的结构可以定义如下：

struct node
{
    int data;
    node *next;
};

利用指针就可以串联起各个节点，实现存储相同类型数据元素的目的。链表的优点是可以快速进行删除和插入操作，只需要改变链表的连接方式，不需要移动大量数据。

但是，访问链表的元素不能使用下标，而是需要从头节点开始遍历整个链表，直到尾节点结束。可以将尾节点的 next 指针设置为 NULL，作为遍历的终止条件。

链表分为单向链表和双向链表。单向链表只能从头节点向后遍历，而双向链表可以从任一方向遍历。特别的，如果尾节点的 next 指针指向头节点，链表就形成了一个环，即单向循环链表。

双向链表中的每个节点除了包含数据外，还包含指向前一个节点和后一个节点的指针。

struct Node {
    int data;
    Node *prev;
    Node *next;
};

双向链表克服了单向链表访问效率较低的缺点，因为它可以从任一方向遍历链表。然而，双向链表需要更多的空间来存储额外的指针。

当然，让我们继续探讨队列。

PS. 栈（Stack）和队列（Queue）

栈是一种**后进先出（LIFO）**的数据结构，类似于现实生活中的堆叠物品。在栈中，最后入栈的元素首先被弹出。栈通常支持两种基本操作：

  1. 入栈（push）：将元素添加到栈顶。
  2. 出栈（pop）：从栈顶移除元素。
  3. 查看栈顶元素（peek）：获取但不移除栈顶的元素。

队列是一种**先进先出（FIFO）**的数据结构，类似于现实生活中排队的情形。在队列中，新元素只能被添加到队尾，而从队列中移除元素时则从只能从队首开始。队列通常支持两种基本操作：

  1. 入队（enqueue）：将元素添加到队列的末尾。
  2. 出队（dequeue）：从队列的开头移除元素。
